Git 问题：git pull 时报错：Connection refused 和 Connection was reset


问题详情：
在本地 git 仓库打开个人博客的代码，几个月没 git pull 了，这时一个 pull 报错了。
使用代理时报错：
1
 
1
fatal: unable to access 'https://github.com/sd445419855/wk-blog.git/': Failed to connect to 127.0.0.1 port 1080 after 1027 ms: Connection refused
不用代理时报错：
1
 
1
fatal: unable to access 'https://github.com/sd445419855/wk-blog.git/': OpenSSL SSL_read: Connection was reset, errno 10054

原因和解法：
经网查和实测，发现 url 没问题，那么推测可能是代理出了问题，解决代理问题后再 pull，显示：
1
 
1
Already up to date.
pull 成功。

那么代理是哪里出了问题？
没有在 1080 端口监听，重启服务就好了，具体原因为知。






Git 问题：git push 时报错：Support for password authentication was removed...


问题描述：
    git pull 连接被拒绝的问题解决后，git push 时又出问题了，报错：
8
 
1
remote: Support for password authentication was removed on August 13, 2021. Please use a personal access token instead.
2
remote: Please see https://github.blog/2020-12-15-token-authentication-requirements-for-git-operations/ for more information.
3
fatal: Authentication failed for 'https://github.com/sd445419855/wk-blog.git/'
4
5
# 翻译：
6
# 远程：2021 年 8 月 13 日移除了对密码身份验证（认证）的支持。请改用个人访问令牌。
7
# 远程：请参阅 https://github.blog/2020-12-15-token-authentication-requirements-for-git-operations/ 了解更多信息。
8
# 致命：“https://github.com/sd445419855/wk-blog.git/”的身份验证失败

问题原因：
为什么要移除我的身份认证呢？想不明白。用第一行错误信息网查发现，是 github 在这一天统一进行了系统的安全升级，以后只能用 token 连，不能用密码连。

问题解法：
既然要用 token，那就去搞 token。搞 token 的方法见《github开发人员在七夕搞事情：remote: Support for password authentication was removed...》。
token 的用法和密码一样，在输入密码的时候输入 token 即可。

注意，github 不会跟你保存 token，需要你自己另存。

为了避免每次 git push 时都要输入账号和密码，我们需要保存账号密码。
git 默认是保存密码的，如果当前设置为不保存密码，那么可以执行下面的命令修改配置为成保存密码：
2
 
1
# 保存账户和密码                                        操作说明：全局配置（文件）中设置保存用户的账号、密码。作用：删除 credential.helper 配置后 git 将不再保存用户账户密码，以便用户可以重新输入账号密码。
2
git config --global --add credential.helper store
