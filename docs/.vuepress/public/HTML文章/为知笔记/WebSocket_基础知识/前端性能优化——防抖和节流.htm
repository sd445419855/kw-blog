概念：



防抖：
2 次调用间隔必须 > 限定间隔，否则延时器重置，无限延时下去；
只最后一次调用有效；
把调用比作抖动：在抖动指令持续发送的情况下，抖动，抖动指令平息一段时间后程序才响应指令抖一下，否则持续等待至间隔时间达到。
当事件快速连续不断触发时，动作只执行 1 次。
有延时器，关键点在延时器重置。


节流：
限定间隔内只调用 1 次；
只第一次调用有效；
把调用比作抖动：在抖动指令持续发送的情况下，隔一段时间就抖一次，间歇等待。
当事件快速连续不断触发时，动作周期性执行。
延时器有的实现有，有的没有，关键点在计算与时间节点的时间差上。

总结：

    防抖、节流只是一种概念，没有固定的实现代码。    
    防抖是延时在执行（延时器），节流是执行在延时（时间差）。
    防抖重在 timer = null，节流重在记录上次执行的时间节点。
    防抖是“目标动作"抖动结束前，目标动作不会执行。（延迟执行，不抖则执行）
    节流是在“目标动作"抖动时，目标动作在一定时间内只让执行一次。 （立即执行，丢帧以降抖）

疑问：

Q：setTimeout 执行完毕后会自动清除吗？
A：setTimeout 和 setInterval 这类方法只是开启一个定时器执行任务，如果任务执行完毕，那么定时器就不工作了，这时候清除不清除定时器就无所谓了。至于使用完毕后不清除是否会占用内存，网上没有明确说法，现代 JS 教程网上也没有对此说明。但可以确定的是，目前没有看你任何一篇文章说定时器执行完任务后必须清除。可见，这并不是一个大问题。
至于生成定时器时返回的定时器 ID，它只是个基本类型的数据，如 Number 类型，就算清除定时器它还是会存在。
至于在极端情况下，排除人为的闭包情况，疯狂 setTimeout 会不会导致内存溢出，这留在以后研究。
