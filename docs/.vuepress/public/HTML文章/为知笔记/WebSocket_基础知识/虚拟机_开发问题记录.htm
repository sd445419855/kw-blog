遇到问题：虚拟机 NAT 模式下 ping 不通宿主机 VMnet8 虚拟网卡


1、问题描述：
虚拟机在 NAT 网络模式下，宿主机可以 ping 通宿主机虚拟网卡 VMnet8，可以 ping 通虚拟机虚拟网卡，而且虚拟机能 ping 通宿主机真实网卡，但邪门的是虚拟机 ping 不通宿主机的虚拟网卡 VMnet8。

2、问题原因：
被 Windows 防火墙墙掉了。

3、解法：
关闭防火墙或在开启防火墙的状态下允许 VMnet8 的流量，可参照这篇文章设置防火墙入栈规则，放行 VMnet8 的网段。 详见：《csdn - windows防火墙 新加虚拟机vmnetwork8》
    
Remark：
其实网上配置网络的教程开头就叫我们关闭主机防火墙了，只是我没当回事。毕竟能 ping 通真机网卡，于是我认为不是防火墙问题，谁知道居然是防火墙放行了 NAT 的流量，但没放行虚拟网卡 VMnet8 的流量。
   放行 VMnet8 流量并不能放行虚拟机 Linux traceroute 接收 ICMP 响应包，解决 traceroute 都是 * 的问题。

4、经验：
    以后开发时最好把所有系统的防火墙都关掉，减少干扰因素，避免走弯路。
