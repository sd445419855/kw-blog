Websocket 消息推送的利弊：


优点 :
开发周期短，维护成本低。
消息不经转第三方服务器，直接由服务器发送到客户端，安全性好。基于 GCM 或者 APNS 的消息推送会把消息发送 GCM 服务器或 APNS 服务器，再由他们转发到客户端。
自己开发服务端，可扩展性好。
对于客户端而言，长连接比轮询的方式性能和及时性更好。

缺点 :
长连接浪费服务端资源。
不能后台运行，一旦 app 退出就不能收到 notification。
由于服务器保持多个长连接，性能将会下降，最大连接数也会有限制。

疑问：

            
Q：能否用 WebSocket 不借助第三方平台和插件实现消息推送？
A：当然可以，第三方平台说白了就是个 WebSocket 服务平台，第三方插件说白了就是一个操作手机通知窗口的 Cordova 插件。
具体做法见当前笔记《Q：websocket 拿到了后台数据后如何将消息搞到用户手机的通知栏上？》


Q：websocket 拿到了后台数据后如何将消息搞到用户手机的通知栏上？
A：其实这个的问题就是在问在 Android 的 WebView 里如何调用系统 API。WebView 要调用系统 API 需要一个中间件 Cordova。
这个 Cordova 插件可以自己手撸，也可以用第三方的 phonegap-plugin-push（注意，这是 PhoneGap的，不是极光的）。
