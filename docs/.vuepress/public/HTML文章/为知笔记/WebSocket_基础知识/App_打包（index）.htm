1.1. 所需的工具和环境：


1.1.1. 所需的工具、环境：

工具：Cordova、Gradle、Ant、Android Studio、NPM
环境：JDK、Android SDK、Node.js




1.1.2. 所需的工具、环境的职责与关系：

Cordova：            HTML 代码转 WebView 代码，使用 npm 所用的源下载 platform（如 Android、iOS）。
Gradle：                依赖包管理工具，含打包功能，类似 Maven。在 Android 里他负责把项目文件打包成 apk。Gradle ≈ Maven + Ant。自 
 cordova-android@4.0.0起，Cordova 使用 Gradle 构建 Android 项目，因此 Gradle 现在是必须安装的。
Ant：                        过时的纯打包的工具，在 Android 里他负责把项目文件打包成 apk。SDK 中把它集成进来了。自 cordova-android@4.0.0起，
Cordova 使用 Gradle 构建 Android 项目，因此 Ant 现在已经不用了，不用安装。
JDK：                      Java 开发环境，打包 apk 时需要它。它并不是 SDK 的依赖，安装它是因为 Android 项目或打包工具里有地方用了 Java 代
码或者 jar 包（如 gradle-wrapper.jar） 。另外，要注意版本，现在 Android 平台要求的 jdk 版本最低是1.8 的。 
Android SDK：    Android 开发环境，打包 apk 时需要。除非你想用  Android Studio，否则并不需要随 Android Studio 一起安装。随 Studio 一
起安装只是可以免去 Studio 对于 SDK 的配置 。
Node.js(含NPM)：以便 Cordova 可以执行 JS 脚本（如 GradleBuilder.js、ProjectBuilder.js）和 npm 命令。
Git(可选)：                cordova 部分功能需要用到 git 命令，比如从 git 仓库下载 platform，但这个不是必须的。不过在当前环境下，需要用 Git 
的 Bash 来执行 xxxx.ps1 脚本文件，使用 powershell 无法执行，而 git bash 上可成功执行，原因不明。

2. App 打包疑问：

Q：SDK Manager 图形版、CMD 版与 Android Studio 的关系?
A：SDK Manager 无论图形版还是 CMD 版本，它都是管理 Android SDK 相关软件包的工具。在 Android Studio 中也有一个 SDK Manager ，但它不叫 SDK Manager ，而是就叫 Android SDK，在 System Settings 里，界面跟独立的 SDK Manager 略微不同，但功能是一样的。最为重要的一点是，它的源是随着 Studio 的版本用着最新的源，不会像独立的停止更新的 SDK Manager 这样还用的老源。
SDK Manager 图形版是老版的 SDK Manager ，新版的 SDK Manager 改成了纯 CMD 版，并且使用的镜像源似乎也发生了改变：
旧源（http://dl.google.com/android/repository/）
新源（https://dl-ssl.google.com/android/repository）
为什么说似乎，是因为这只是一种推测，推测的来由是从旧源下载的有的包不是配置文件要求的包（如 DHU）。
官方只推行新的 CMD 版本，旧的图形版本不再做维护和更新。
