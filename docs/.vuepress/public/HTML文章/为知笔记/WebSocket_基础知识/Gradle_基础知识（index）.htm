build.gradle 的概念


图 - 根目录下的 build.gradle

图 - app 目录下的 build.gradle

Android 项目中用 Gradle 构建 App，这样的项目中会有 3 个或者更多的 build.gradle，一个在根目录下，一个在 app 目录下，一个在 CordovaLib 目录下。

根（android）目录下的 build.gradle：
主要用于 Gradle 构建一个 App 时，为其指定依赖（如 具体的构建 App 用的插件，com.android.tools.build:gradle:3.0.1）和依赖的远程仓库（如  url "https://maven.google.com"）

app 目录下的 build.gradle：
主要用于 Gradle 构建一个 App 时，为其指定构建 App 的详细配置，比如声明当前构建的项目是一个 App 还是一个模块。另外，其中包括指定依赖和依赖的远程仓库。

CordovaLib 目录下的 build.gradle：
主要用于 Gradle 构建一个 App 时，为其指定构建 App 里的依赖 CordovaLib 的详细配置，比如像根（android）目录下的 build.gradle 一样再一次指定依赖（如 具体的构建 App 用的插件，com.android.tools.build:gradle:3.0.1）和依赖的远程仓库（如  url "https://maven.google.com"）。

Tips：
build.gradle 里 “buildscript {}” 形式的声明都是闭包。

Q：jCenter 到底是 Maven 那样的仓库还是 GitHub 那样的仓库？        
A：与 Maven 仓库相同性质、功能的类库文件的仓库（Library，依赖包仓库），提供 Java 和 Android 的库文件。 Maven 和 jCenter 都可以是个人上传依赖
包，比如“阿里短信通知功能“，只是需要审核。仓库上可以上传库文件、源码、文档、pom 等，只是源码的存在形式和 GitHub 不同，它不像 GitHub 那样是展开的，得打包成一个压缩文件，详见《上传项目到JCenter》中 jCenter 的文件列表。
上传类库文件到 jCenter 还是 Maven，取决于开发者。

Q：把开源项目发布到 jCenter，是把源码像 Github 一样发布上去吗？
A：当然不是，是像 Maven 仓库那样。

Q：在 Gradle 构建项目时引用开源项目，引用源码吗？不用下载吗？            
A：源码个锤子，源码要整个编译才能执行，你单单引用源码有锤子用，得引用编译、打包后的二进制类库文件，如 jar，而不是 zip。
详见《上传项目到JCenter》。

Q：Gradle 构建项目时从 jCenter 获取构件，这个构件是什么？            
A：就是 component，像 jar 包那样的东西。详见《百度 - 构件 - 软件工程》
